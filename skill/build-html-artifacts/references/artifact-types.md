# Artifact types

## Dashboards

Lead with the decision the dashboard supports. Put a concise status summary before metrics. Combine KPIs with context, comparisons, and exceptions. Provide filtering, freshness, and data-source information. Design loading, empty, partial, and error states.

State the supported decision as a headline, subhead, or compact unboxed statement. Do not automatically turn it into an alert, callout, tinted card, or edge-accented panel. Use a notice component only when the content is a genuine exception, warning, approval request, or required action.

## Narrative documents

Prioritize reading rhythm, hierarchy, figures, citations, and print output. Use a restrained measure, useful table of contents, descriptive figure captions, and page-break rules. Let evidence interrupt the narrative only when it advances the argument.

## Presentations

Build discrete full-viewport sections with clear progression. Support keyboard navigation, visible slide position, and print-to-PDF. Use one idea per slide and keep essential content inside safe margins at common aspect ratios.

Default to collapsed presentation controls: show only a compact slide-position button at an edge, then disclose previous/next controls and clickable slide dots on click. Each dot must reveal its slide title on hover and keyboard focus, indicate the current slide, and jump directly to its slide when selected. Collapse the panel again after a navigation choice, support Escape to close it, and keep arrow-key navigation available while it is hidden. Use a persistent bar or sidebar only for nonlinear reference decks or an explicit request for always-visible section navigation. Keep presentation chrome visually subordinate to the current slide and never reserve a full-width strip for controls by default.

Treat each slide conclusion as part of the editorial hierarchy: a decisive headline, a short rule-and-text treatment, or a compact closing statement. Do not automatically turn conclusions, insights, or takeaways into tinted callout boxes. Reserve information icons and semantic callouts for actual notices, risks, exceptions, or actions, and avoid repeating the same boxed treatment on every slide.

Never use a thick colored leading border as the rule-and-text treatment. If a rule is useful, keep it neutral, thin, and separated from the text rather than attached to a rounded container.

## Operational interfaces

Optimize for repeated use, density, predictable placement, and recovery from errors. Preserve user context during filtering and updates. Make destructive actions explicit and reversible when possible.

## Microsites

Create a strong opening proposition and a deliberate narrative sequence. Keep navigation shallow, calls to action specific, and performance independent of ornamental media.
